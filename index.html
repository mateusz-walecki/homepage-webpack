Przy otwarciu czystego dokumentu zacznij od:

  1. W arkuszu html wpisz znak "!" a następnie tab.
    • Po tym pamiętaj by zmnienić język w jakim będzie wyświetlany tekst strony, tą możliwość znajdziesz tutaj: < html lang="pl"> domyślnie nie będzie to "pl"
    • W nagłówku < head > podaj ścieżkę do plików formatujących:
      <link href="normalize.css" rel="stylesheet" type="text/css">
  2. Zakładamy nowe repozytorium na GitHub
  3. Tworzymy pliki w katalogu np:
    • index.html
    • style.css
    • main.js
    • normalize.css
    • main.scss
  4. W konsoli wpisujemy:
    • Ścieżkę do katalogu na kompie (np. cd C:\dev\homepage)
    • git init (zakładanie gita w katalogu)
    • dir
    • git status
    • git add .
    • git commit -m "..."
    • Wklejamy komendę z GitHub
    • git push
  5. Instalujemy jednorazowo paczkę node. W tym celu wchodzimy na nodejs.org i pobieramy rekomendowaną paczkę, instalujemy.
  6. Po instalacji w konsoli wpisujemy:
    • npm init -y
    • npm install node-sass --save-dev (paczka node sass, równiez instalacja jednorazowa)
  7. Utwórz plik .gitignore w katalogu
    • dodaj w nim node_modules
  8. By pliki np main.scss zamieniał kod w main.css wpisz w konsoli:
    • npx node-sass main.scss main.css
    • od teraz pisz kod wyłącznie w .scss!!! nie edytuj .css!!!
    • W pliku package.json pod "scripts:"main.js dodaj:
    • "sass": "npx node-sass -w style.scss style.css",
    • Gdybyś miał ów plik w podkatalogu wtedy piszesz:
    • "sass": "npx node-sass -w nazwa podkatalogu/main.scss main.css",
    • Wpisz w konsoli komendę: npm run sass by sprwdzić czy zadziałało Uruchom komendę za każdym razem gdy działasz na repozytorium!!!
  9. Stwórz plik normalize.css bądż zaimportuj go z zewnątrz (VII/2)

WEBPACK

  1. Kopiowanie pakietu webpack z strony WTF (VII.05)
    • Mamy dostępny taki pakiet od WTF pod : linkiem, pożniej oczywiście creating repozytory
    • Skopiuj repozytorium tak jak zawsze z gita i z pomocą CMD ściągnij go do siebie. Ścieżka do nowego repa -> git clone [adres z gita]
    • cd {nazwa katalogu}
    • odpal edytor w cmd-> code .
    • npm install
    • npm run start
    • Po tym standardowo powinieneś otworzyć stronę wpisując w przeglądarce standardowy adres: localhost:8080
    • By wyjść z trybu kliknij ctrl+c
    • ponowne wejście to npm run start
    • doinstaluj katalog dist w cmd: npm run build
  2. Ścieżki do plików

    3. Podajemy je względem plików źródłowych (nie serwerowych).

    • W katalogu src-> pages mamy pliki stron. W nich (np index.html) w części body->div podejemy adresy do plików.
    • Webpack sam znajdzie sobie ten plik dzięki wtyczce do html-a i spokiuje go do katalogu dist. Podczas kopiowania zmieniu mu nazwę aby ten plik był zawszę na "świeżo", by przeglądarka nie zapisała sobie starej wersji, pozwala to na poprawne ładownie strony
    • Obrazki zawszę wrzucamy do katalogu assets-> img
  3. Dodawanie plików
    • Gdy w folderze pages dodamy stronę (np: mateusz.html) musimy w pliku webpack.commoj.js skopiować new HtmlWebpackPlugin i w ścieżce podać nazwę pliku.
    • w części filname zmień też nazwę pliku
    • Zrestartuj serwer z pomocą: ctrl+c a następnie npm run start tak by widzieć stronę w przeglądarce
    • By plik nie brał komend JS które wykonywane są na innych stronach utwórz nowy plik w src-> js
    • Następnie w pliku webpack.commoj.js w części entry: dodaj nasz plik html oraz ścieżkę do JS
    • Później w new HtmlWebpackPlugin w części chunks tuż po "index" podejesz nazwe pliku JS
  4. GitHub
    • Pushing na Gita
    • By uruchomić strone w GitHub-ie wchodzimy
      • W CMD wpisujemy komende: npm run publish
      • Ustawienia-> Sekcja GitHub Pages-> rozwijasz liste i wybierasz: gh-pages branch
      • Otwórz link poniżej (poczekaj chwile aż się załaduje)

Struktura pisania w HTML- początek

Ogólnie html jest prosty jak wydanie 2mld zł na tvp

Podstawowa wiedza:

< body > wszystko co chcemy wyswietlic na stronie musi być umieszczone w tym znaczniku

< main > jest to główna część/treść naszej strony, ona stanowi priorytet przy wyświetlaniu na przeglądarkach

< header > może stanowić nagłówek dokumentu, logo itp, czyli element mający rzucać się w oczy, wewnątrz niego umieszczamy zazwyczaj < h1 > NAGŁÓWEK < /h1 >

-dodając header pod main będzie on rozpatrywany przez przeglądarkę

-dodając go przed < /main > wyszukiwarki nie będą go analizować, przykładowo można go użyć jako przyscisk powrotu do strony głównej

< section > sekcje dokumentu (części podziału strony)

< article > samodzielna treść/część strony

< h1 > nagłówek pierwszego poziomu, najniższym jest h6

< p > paragraf/akapit, tekst główny umieszczony np. pod nagłówkami

< footer > stopka dokumentu/strony, najlepiej przed < /main > tak jak w przypadku header
